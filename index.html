HOW TO Close PowerShell and open it fresh ▼

You'll be told to do this several times. Here's exactly what it means:

  1. Click the X button in the top-right corner of the PowerShell window to close it completely
  2. Press the Windows key on your keyboard
  3. Type PowerShell and press Enter
  4. You now have a fresh window — continue with the next step
NOTEA fresh window is important because Windows only loads new settings when a window first opens. Changes don't take effect until you reopen it.
HOW TO Find your Windows username ▼

Several steps ask you to replace YourName with your actual Windows username. Here's how to find it:

  1. Open PowerShell
  2. Type echo $env:USERNAME and press Enter
  3. The result that appears is your username — for example NatalieTrent

Everywhere you see YourName in this guide, replace it with that exact result. For example if your username is NatalieTrent, then C:\Users\YourName becomes C:\Users\NatalieTrent.

PART 1 — PREREQUISITES
1
Open PowerShell and allow scripts SETUP ▲

PowerShell is the text window where you type all the commands in this guide.

  1. Press the Windows key
  2. Type PowerShell
  3. Click Windows PowerShell — the plain one, NOT "Run as Administrator"
  4. When it opens, type the command below exactly and press Enter:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

When it asks a question, type Y and press Enter.

IMPORTANTNever open PowerShell as Administrator unless this guide specifically says so.
2
Install Git INSTALL ▲

Git is required before Claude Code will work. Type this and press Enter:

winget install Git.Git

When it finishes, close PowerShell and open it fresh (see the "How to" box above). Then type this to confirm it worked:

git --version
SUCCESSYou should see something like: git version 2.49.0.windows.1 — any version number means it worked.
IMPORTANTIf you see "git is not recognized" — close PowerShell and open it fresh again. This solves it almost every time.

Now find exactly where Git was installed — you'll need this in Step 6:

where.exe git
NOTEWrite down the result. It will look like C:\Users\NatalieTrent\AppData\Local\Programs\Git\cmd\git.exe — keep it handy. You need it in Step 6. You can also paste it into the "Build your personal commands" tool above.
3
Install Node.js INSTALL ▲

Node.js is required by several other tools. Type this and press Enter:

winget install OpenJS.NodeJS.LTS --scope user

Close PowerShell and open it fresh. Then verify both of these:

node --version
npm --version
SUCCESSYou should see two version numbers — one for each command. That's all you need.
4
Install VS Code (code editor) INSTALL ▲

VS Code is where you'll read and review what Claude writes. You don't need to write code yourself.

  1. Open your web browser and go to code.visualstudio.com
  2. Click Download for Windows
  3. Run the installer — on "Select Additional Tasks," check all boxes
  4. Finish installation and open VS Code

Inside VS Code, press Ctrl+Shift+X to open Extensions. Search for and install each of these:

  • Prettier
  • ESLint
  • GitLens
  • GitHub Pull Requests
NOTEDo NOT install the Claude Code extension yet — install it after Step 6.
5
Set up GitHub and GitHub CLI SETUP ▲

GitHub stores your team's work online. GitHub CLI prevents password errors when saving your work.

  1. Open your browser and go to github.com
  2. Create a free account and verify your email

Back in PowerShell, install GitHub CLI:

winget install GitHub.cli

Close PowerShell, open it fresh, then log in:

gh auth login

It will ask questions. Answer like this:

  • Where do you use GitHub? → choose GitHub.com
  • Protocol? → choose HTTPS
  • Authenticate with browser? → type Y and press Enter
  • Complete the login in the browser that opens, then come back to PowerShell
SUCCESSAfter this you'll never be asked for a GitHub password in the terminal again.
PART 2 — INSTALL CLAUDE CODE
6
Install Claude Code — read the whole step before starting INSTALL ▲
IMPORTANTThis is where most installs go wrong. Read all the way through before typing anything.

First — build your Git Bash path

In Step 2 you ran where.exe git and got a path ending in cmd\git.exe. Change the end of that path from cmd\git.exe to bin\bash.exe.

What you have: C:\Users\NatalieTrent\AppData\Local\Programs\Git\cmd\git.exe
↓ Change the end to get:
What you need: C:\Users\NatalieTrent\AppData\Local\Programs\Git\bin\bash.exe
NOTEIf you filled in the "Build your personal commands" tool above, your bash path is already shown there — copy it from there.

Second — set the path variable (do this BEFORE the installer)

Replace the orange part with your actual bash path:

# Replace the orange part with YOUR actual bash path
$env:CLAUDE_CODE_GIT_BASH_PATH = "C:\Users\YourName\AppData\Local\Programs\Git\bin\bash.exe"
IMPORTANTYou MUST run this line BEFORE the installer below. If you skip it, the install looks successful but won't actually work.

Third — run the installer

irm https://claude.ai/install.ps1 | iex
SUCCESSA successful install shows: ✔ Claude Code successfully installed! and a location ending in \.local\bin\claude.exe

Fourth — make the settings permanent

Run both commands below. Replace orange parts with your username and your bash path:

# Makes claude available every time you open PowerShell
[Environment]::SetEnvironmentVariable("PATH", "C:\Users\YourName\.local\bin;" + [Environment]::GetEnvironmentVariable("PATH", "User"), "User")
# Saves your Git Bash path permanently
[Environment]::SetEnvironmentVariable("CLAUDE_CODE_GIT_BASH_PATH", "C:\Users\YourName\AppData\Local\Programs\Git\bin\bash.exe", "User")

Fifth — restart PowerShell and verify

Close PowerShell completely and open it fresh. Then type:

claude --version
SUCCESSYou should see a version number like claude 2.1.78. Claude Code is fully installed.
7
Log in to Claude Code (one time) SETUP ▲

Create a folder for your work and link Claude to your Anthropic account. Type each line one at a time, pressing Enter after each. Replace the orange parts with your Windows username:

mkdir C:\Users\YourName\Projects
cd C:\Users\YourName\Projects
mkdir my-first-app
cd my-first-app
git init
claude

After you type claude and press Enter, your web browser will open automatically to claude.ai. Sign in with your Anthropic account. Come back to PowerShell — you'll see the Claude Code welcome screen.

NOTEYour login is saved. You won't need to do this again on this computer.

PART 3 — YOUR FIRST PROJECT
10
How to start working with Claude each day DAILY USE ▲

Every time you want to work on a project with Claude, do these three things:

Step A — Open PowerShell

  1. Press the Windows key
  2. Type PowerShell and press Enter

Step B — Navigate to your project folder

Replace the orange parts with your username and project folder name:

cd C:\Users\YourName\Projects\your-project-name
NOTENot sure what to type for the project name? Open File Explorer and go to C:\Users\YourName\Projects — the folder names there are what you'd type.

Step C — Start Claude

claude

Claude will start up and show you a prompt. Type your instructions in plain English and press Enter. Claude will respond and ask for your approval before changing anything.

NOTEClaude always asks before making changes. Read what it proposes, then type yes or no.
11
How to start a brand new project DAILY USE ▲

Use this when starting something completely new. Replace the orange part with your project name — no spaces, use dashes instead (e.g. compliance-tracker):

cd C:\Users\YourName\Projects
mkdir my-project-name
cd my-project-name
git init
claude

Once Claude starts, introduce your project. Example opening message:

"I'm starting a new project. It's a [describe what it does in plain English]. I need you to help me plan and build it from scratch. Please ask me questions so you understand what I need before you start writing any code."
12
Phrases to use with Claude — saving work and handing off DAILY USE ▲

These are the key phrases your team needs to know. Say them exactly like this inside a Claude session by typing them and pressing Enter.

SAY THIS"commit everything and give me a handoff summary"
When: end of your work session, or before you close PowerShell
This tells Claude to save all your work to Git with a description of what was done, then give you a written summary of where things stand so you or a teammate can pick up exactly where you left off. Always do this before closing PowerShell.
SAY THIS"what's the status of this project?"
When: starting a session and you want a recap
Claude will read your project files and give you a plain-English summary of what exists, what's done, and what still needs to happen.
SAY THIS"pick up where we left off"
When: continuing work from a previous session
Say this after starting Claude in your project folder. Claude will review the recent history and handoff notes and resume from where you stopped.
SAY THIS"don't change anything yet — just tell me your plan first"
When: before any big task
Claude will describe every step it plans to take and wait for your approval before touching a single file. Use this any time you're not sure what Claude is about to do.
SAY THIS"save a checkpoint"
When: something is working and you don't want to lose it
Claude will commit your current working state to Git so you can always get back to this point, even if something breaks later.
SAY THIS"undo the last change"
When: Claude did something you didn't want
Claude will revert the most recent change to your files. If it can't undo automatically, it will tell you exactly which files to restore.
SAY THIS"explain what this does in plain English"
When: you see code or output you don't understand
Claude will translate anything technical into plain language. You can point at a specific file, a section of code, or just ask about the project generally.
SAY THIS"something broke — help me figure out what happened"
When: anything stops working
Paste any error message after this phrase. Claude will read the error and your project files and tell you exactly what went wrong and how to fix it.

PART 4 — TROUBLESHOOTING
Common problems and fixes
"claude" is not recognized
Close PowerShell completely (click X) and open a fresh window. This fixes it 90% of the time.
"Unable to find CLAUDE_CODE_GIT_BASH_PATH"
Your bash path is wrong. Re-run Step 2 (where.exe git), rebuild the path changing cmd\git.exe to bin\bash.exe, and redo Step 6.
"npm" or "node" is not recognized
Close PowerShell and open it fresh after installing Node.js in Step 3.
"git" is not recognized
Close and reopen PowerShell. If still not working, re-run Step 2.
"running scripts is disabled"
Re-run the Set-ExecutionPolicy command from Step 1.
"Could not remove temporary file" warning
Harmless — just a leftover download file. Ignore it and continue.
git push asks for a password
Run gh auth login (Step 5). GitHub no longer accepts passwords.
Install looks done but claude still doesn't work
Run both SetEnvironmentVariable commands in Step 6 again, then close and reopen PowerShell.
I closed PowerShell without committing
Reopen PowerShell, cd back to your project folder, type claude, then say "what's the status of this project?" Claude can see uncommitted files.
