HOW TO Close PowerShell and open it fresh ▼
  1. Click the X button in the top-right corner of the PowerShell window to close it completely
  2. Press the Windows key on your keyboard
  3. Type PowerShell and press Enter
  4. You now have a fresh window — continue with the next step
NOTEA fresh window is important because Windows only loads new settings when a window first opens.

  1. Open PowerShell

  3. The result is your username — e.g. Natalie

PART 1 — PREREQUISITES
1
Open PowerShell and allow scripts
SETUP
▼

PowerShell is the text window where you type all the commands in this guide.

  1. Press the Windows key
  2. Type PowerShell
  3. Click Windows PowerShell — the plain one, NOT "Run as Administrator"
  4. When it opens, type the command below exactly and press Enter:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

When it asks a question, type Y and press Enter.

IMPORTANTNever open PowerShell as Administrator unless this guide specifically says so.
2
Install Git
INSTALL
▼

Git is required before Claude Code will work. Type this and press Enter:

winget install Git.Git

When it finishes, close PowerShell and open it fresh. Then type this to confirm it worked:

git --version
SUCCESSYou should see something like: git version 2.49.0.windows.1 — any version number means it worked.
IMPORTANTIf you see "git is not recognized" — close PowerShell and open it fresh again. This solves it almost every time.

Now find exactly where Git was installed — you'll need this in Step 6:

where.exe git
NOTECopy the result and paste it into the "Your Git path" field in the tool above. The guide will auto-build your bash path and pre-fill Step 6 for you.
3
Install Node.js
INSTALL
▼

Node.js is required by several other tools. Type this and press Enter:

winget install OpenJS.NodeJS.LTS --scope user

Close PowerShell and open it fresh. Then verify both of these:

node --version
npm --version
SUCCESSYou should see two version numbers — one for each command.
4
Install VS Code (code editor)
INSTALL
▼

VS Code is where you'll read and review what Claude writes. You don't need to write code yourself.

  1. Open your web browser and go to code.visualstudio.com
  2. Click Download for Windows
  3. Run the installer — on "Select Additional Tasks," check all boxes
  4. Finish installation and open VS Code

Inside VS Code, press Ctrl+Shift+X to open Extensions. Search for and install each of these:

  • Prettier
  • ESLint
  • GitLens
  • GitHub Pull Requests
NOTEDo NOT install the Claude Code extension yet — install it after Step 6.
5
Set up GitHub and GitHub CLI
SETUP
▼

GitHub stores your team's work online. GitHub CLI prevents password errors when saving your work.

  1. Open your browser and go to github.com
  2. Create a free account and verify your email

Back in PowerShell, install GitHub CLI:

winget install GitHub.cli

Close PowerShell, open it fresh, then log in:

gh auth login

It will ask questions. Answer like this:

  • Where do you use GitHub? → choose GitHub.com
  • Protocol? → choose HTTPS
  • Authenticate with browser? → type Y and press Enter
  • Complete the login in the browser that opens, then come back to PowerShell
SUCCESSAfter this you'll never be asked for a GitHub password in the terminal again.
PART 2 — INSTALL CLAUDE CODE
6
Install Claude Code — read the whole step before starting
INSTALL
▼
IMPORTANTThis is where most installs go wrong. Read all the way through before typing anything.

First — your Git Bash path

In Step 2 you ran where.exe git and got a path ending in cmd\git.exe. Change the end from cmd\git.exe to bin\bash.exe.

NOTEIf you filled in the tool at the top of the page, your bash path is already shown there — and the commands below are pre-filled for you.

Second — set the path variable (do this BEFORE the installer)

# Replace the path with YOUR actual bash path if not pre-filled $env:CLAUDE_CODE_GIT_BASH_PATH = "C:\Users\YourName\AppData\Local\Programs\Git\bin\bash.exe"
IMPORTANTYou MUST run this line BEFORE the installer below. If you skip it, the install looks successful but won't actually work.

Third — run the installer

irm https://claude.ai/install.ps1 | iex
SUCCESSA successful install shows: ✔ Claude Code successfully installed! and a location ending in \.local\bin\claude.exe

Fourth — make the settings permanent

Run both commands below. They are pre-filled with your details if you used the tool above:

# Makes claude available every time you open PowerShell [Environment]::SetEnvironmentVariable("PATH", "C:\Users\YourName\.local\bin;" + [Environment]::GetEnvironmentVariable("PATH", "User"), "User")
# Saves your Git Bash path permanently [Environment]::SetEnvironmentVariable("CLAUDE_CODE_GIT_BASH_PATH", "C:\Users\YourName\AppData\Local\Programs\Git\bin\bash.exe", "User")

Fifth — restart PowerShell and verify

Close PowerShell completely and open it fresh. Then type:

claude --version
SUCCESSYou should see a version number like claude 2.1.78. Claude Code is fully installed.
7
Log in to Claude Code (one time)
SETUP
▼

Create a folder for your work and link Claude to your Anthropic account. Type each line one at a time, pressing Enter after each:

mkdir C:\Users\YourName\Projects
cd C:\Users\YourName\Projects
mkdir my-first-app
cd my-first-app
git init
claude

After you type claude and press Enter, your web browser will open automatically to claude.ai. Sign in with your Anthropic account. Come back to PowerShell — you'll see the Claude Code welcome screen.

NOTEYour login is saved. You won't need to do this again on this computer.

IMPORTANTThis extension only works if Steps 6 and 7 are complete first.

PART 3 — YOUR FIRST PROJECT
10
How to start working with Claude each day
DAILY USE
▼

Every time you want to work on a project with Claude, do these three things:

Step A — Open PowerShell

  1. Press the Windows key
  2. Type PowerShell and press Enter

Step B — Navigate to your project folder

Replace your-project-name with the actual folder name:

cd C:\Users\YourName\Projects\your-project-name
NOTENot sure what to type for the project name? Open File Explorer and go to C:\Users\YourName\Projects — the folder names there are what you'd type.

Step C — Start Claude

claude
NOTEClaude always asks before making changes. Read what it proposes, then type yes or no.
11
How to start a brand new project
DAILY USE
▼

Use this when starting something completely new. Replace my-project-name with your project name — no spaces, use dashes instead (e.g. compliance-tracker):

cd C:\Users\YourName\Projects
mkdir my-project-name
cd my-project-name
git init
claude

Once Claude starts, introduce your project. Example opening message:

"I'm starting a new project. It's a [describe what it does in plain English]. I need you to help me plan and build it from scratch. Please ask me questions so you understand what I need before you start writing any code."
12
Phrases to use with Claude — saving work and handing off
DAILY USE
▼

These are the key phrases your team needs to know. Say them exactly like this inside a Claude session.

Say thisWhen to use it
commit everything and give me a handoff summaryEnd of your work session, or before you close PowerShell. Always do this before closing.
what's the status of this project?Starting a session and want a recap of where things stand.
pick up where we left offContinuing work from a previous session.
don't change anything yet — just tell me your plan firstBefore any big task — Claude will outline steps and wait for your approval.
save a checkpointSomething is working and you don't want to lose it.
undo the last changeClaude did something you didn't want.
explain what this does in plain EnglishYou see code or output you don't understand.
something broke — help me figure out what happenedAnything stops working. Paste any error message after this phrase.

PART 4 — TROUBLESHOOTING
?
Common problems and fixes
▼
"claude" is not recognizedClose PowerShell completely (click X) and open a fresh window. This fixes it 90% of the time.
Unable to find CLAUDE_CODE_GIT_BASH_PATHYour bash path is wrong. Re-run Step 2 (where.exe git), update the tool at the top of this page, and redo Step 6.
"npm" or "node" is not recognizedClose PowerShell and open it fresh after installing Node.js in Step 3.
"git" is not recognizedClose and reopen PowerShell. If still not working, re-run Step 2.
running scripts is disabledRe-run the Set-ExecutionPolicy command from Step 1.
"Could not remove temporary file" warningHarmless — just a leftover download file. Ignore it and continue.
git push asks for a passwordRun gh auth login (Step 5). GitHub no longer accepts passwords.
Install looks done but claude still doesn't workRun both SetEnvironmentVariable commands in Step 6 again, then close and reopen PowerShell.
I closed PowerShell without committingReopen PowerShell, cd back to your project folder, type claude, then say "what's the status of this project?" Claude can see uncommitted files.
